      • shouldPersistStateChange

        private boolean shouldPersistStateChange(HRegionInfo hri,
                               RegionState state,
                               RegionState oldState)
        Check if we should persist a state change in meta. Generally it's better to persist all state changes. However, we should not do that if the region is not in meta at all. Based on the state and the previous state, we can identify if a user region has an entry in meta. For example, merged regions are deleted from meta; New merging parents, or splitting daughters are not created in meta yet.

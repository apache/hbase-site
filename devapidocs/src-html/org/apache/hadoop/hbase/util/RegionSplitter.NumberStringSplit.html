070/**
071 * The {@link RegionSplitter} class provides several utilities to help in the
072 * administration lifecycle for developers who choose to manually split regions
073 * instead of having HBase handle that automatically. The most useful utilities
074 * are:
075 * <p>
076 * <ul>
077 * <li>Create a table with a specified number of pre-split regions
078 * <li>Execute a rolling split of all regions on an existing table
079 * </ul>
080 * <p>
081 * Both operations can be safely done on a live server.
082 * <p>
083 * <b>Question:</b> How do I turn off automatic splitting? <br>
084 * <b>Answer:</b> Automatic splitting is determined by the configuration value
085 * <i>HConstants.HREGION_MAX_FILESIZE</i>. It is not recommended that you set this
086 * to Long.MAX_VALUE in case you forget about manual splits. A suggested setting
087 * is 100GB, which would result in &gt; 1hr major compactions if reached.
088 * <p>
089 * <b>Question:</b> Why did the original authors decide to manually split? <br>
090 * <b>Answer:</b> Specific workload characteristics of our use case allowed us
091 * to benefit from a manual split system.
092 * <p>
093 * <ul>
094 * <li>Data (~1k) that would grow instead of being replaced
095 * <li>Data growth was roughly uniform across all regions
096 * <li>OLTP workload. Data loss is a big deal.
097 * </ul>
098 * <p>
099 * <b>Question:</b> Why is manual splitting good for this workload? <br>
100 * <b>Answer:</b> Although automated splitting is not a bad option, there are
101 * benefits to manual splitting.
102 * <p>
103 * <ul>
104 * <li>With growing amounts of data, splits will continually be needed. Since
105 * you always know exactly what regions you have, long-term debugging and
106 * profiling is much easier with manual splits. It is hard to trace the logs to
107 * understand region level problems if it keeps splitting and getting renamed.
108 * <li>Data offlining bugs + unknown number of split regions == oh crap! If an
109 * WAL or StoreFile was mistakenly unprocessed by HBase due to a weird bug and
110 * you notice it a day or so later, you can be assured that the regions
111 * specified in these files are the same as the current regions and you have
112 * less headaches trying to restore/replay your data.
113 * <li>You can finely tune your compaction algorithm. With roughly uniform data
114 * growth, it's easy to cause split / compaction storms as the regions all
115 * roughly hit the same data size at the same time. With manual splits, you can
116 * let staggered, time-based major compactions spread out your network IO load.
117 * </ul>
118 * <p>
119 * <b>Question:</b> What's the optimal number of pre-split regions to create? <br>
120 * <b>Answer:</b> Mileage will vary depending upon your application.
121 * <p>
122 * The short answer for our application is that we started with 10 pre-split
123 * regions / server and watched our data growth over time. It's better to err on
124 * the side of too little regions and rolling split later.
125 * <p>
126 * The more complicated answer is that this depends upon the largest storefile
127 * in your region. With a growing data size, this will get larger over time. You
128 * want the largest region to be just big enough that the
129 * {@link org.apache.hadoop.hbase.regionserver.HStore} compact
130 * selection algorithm only compacts it due to a timed major. If you don't, your
131 * cluster can be prone to compaction storms as the algorithm decides to run
132 * major compactions on a large series of regions all at once. Note that
133 * compaction storms are due to the uniform data growth, not the manual split
134 * decision.
135 * <p>
136 * If you pre-split your regions too thin, you can increase the major compaction
137 * interval by configuring HConstants.MAJOR_COMPACTION_PERIOD. If your data size
138 * grows too large, use this script to perform a network IO safe rolling split
139 * of all regions.
140 */

145  /**
146   * A generic interface for the RegionSplitter code to use for all it's
147   * functionality. Note that the original authors of this code use
148   * {@link HexStringSplit} to partition their table and set it as default, but
149   * provided this for your custom algorithm. To use, create a new derived class
150   * from this interface and call {@link RegionSplitter#createPresplitTable} or
151   * RegionSplitter#rollingSplit(TableName, SplitAlgorithm, Configuration) with the
152   * argument splitClassName giving the name of your class.
153   */
154  public interface SplitAlgorithm {
155    /**
156     * Split a pre-existing region into 2 regions.
157     *
158     * @param start
159     *          first row (inclusive)
160     * @param end
161     *          last row (exclusive)
162     * @return the split row to use
163     */
164    byte[] split(byte[] start, byte[] end);
165
166    /**
167     * Split an entire table.
168     *
169     * @param numRegions
170     *          number of regions to split the table into
171     *
172     * @throws RuntimeException
173     *           user input is validated at this time. may throw a runtime
174     *           exception in response to a parse failure
175     * @return array of split keys for the initial regions of the table. The
176     *         length of the returned array should be numRegions-1.
177     */
178    byte[][] split(int numRegions);
179
180    /**
181     * Some MapReduce jobs may want to run multiple mappers per region,
182     * this is intended for such usecase.
183     *
184     * @param start first row (inclusive)
185     * @param end last row (exclusive)
186     * @param numSplits number of splits to generate
187     * @param inclusive whether start and end are returned as split points
188     */
189    byte[][] split(byte[] start, byte[] end, int numSplits, boolean inclusive);
190
191    /**
192     * In HBase, the first row is represented by an empty byte array. This might
193     * cause problems with your split algorithm or row printing. All your APIs
194     * will be passed firstRow() instead of empty array.
195     *
196     * @return your representation of your first row
197     */
198    byte[] firstRow();
199
200    /**
201     * In HBase, the last row is represented by an empty byte array. This might
202     * cause problems with your split algorithm or row printing. All your APIs
203     * will be passed firstRow() instead of empty array.
204     *
205     * @return your representation of your last row
206     */
207    byte[] lastRow();
208
209    /**
210     * In HBase, the last row is represented by an empty byte array. Set this
211     * value to help the split code understand how to evenly divide the first
212     * region.
213     *
214     * @param userInput
215     *          raw user input (may throw RuntimeException on parse failure)
216     */
217    void setFirstRow(String userInput);
218
219    /**
220     * In HBase, the last row is represented by an empty byte array. Set this
221     * value to help the split code understand how to evenly divide the last
222     * region. Note that this last row is inclusive for all rows sharing the
223     * same prefix.
224     *
225     * @param userInput
226     *          raw user input (may throw RuntimeException on parse failure)
227     */
228    void setLastRow(String userInput);
229
230    /**
231     * @param input
232     *          user or file input for row
233     * @return byte array representation of this row for HBase
234     */
235    byte[] strToRow(String input);
236
237    /**
238     * @param row
239     *          byte array representing a row in HBase
240     * @return String to use for debug &amp; file printing
241     */
242    String rowToStr(byte[] row);
243
244    /**
245     * @return the separator character to use when storing / printing the row
246     */
247    String separator();
248
249    /**
250     * Set the first row
251     * @param userInput byte array of the row key.
252     */
253    void setFirstRow(byte[] userInput);
254
255    /**
256     * Set the last row
257     * @param userInput byte array of the row key.
258     */
259    void setLastRow(byte[] userInput);
260  }

262  /**
263   * The main function for the RegionSplitter application. Common uses:
264   * <p>
265   * <ul>
266   * <li>create a table named 'myTable' with 60 pre-split regions containing 2
267   * column families 'test' &amp; 'rs', assuming the keys are hex-encoded ASCII:
268   * <ul>
269   * <li>bin/hbase org.apache.hadoop.hbase.util.RegionSplitter -c 60 -f test:rs
270   * myTable HexStringSplit
271   * </ul>
272   * <li>create a table named 'myTable' with 50 pre-split regions,
273   * assuming the keys are decimal-encoded ASCII:
274   * <ul>
275   * <li>bin/hbase org.apache.hadoop.hbase.util.RegionSplitter -c 50
276   * myTable DecimalStringSplit
277   * </ul>
278   * <li>perform a rolling split of 'myTable' (i.e. 60 =&gt; 120 regions), # 2
279   * outstanding splits at a time, assuming keys are uniformly distributed
280   * bytes:
281   * <ul>
282   * <li>bin/hbase org.apache.hadoop.hbase.util.RegionSplitter -r -o 2 myTable
283   * UniformSplit
284   * </ul>
285   * </ul>
286   *
287   * There are three SplitAlgorithms built into RegionSplitter, HexStringSplit,
288   * DecimalStringSplit, and UniformSplit. These are different strategies for
289   * choosing region boundaries. See their source code for details.
290   *
291   * @param args
292   *          Usage: RegionSplitter &lt;TABLE&gt; &lt;SPLITALGORITHM&gt;
293   *          &lt;-c &lt;# regions&gt; -f &lt;family:family:...&gt; | -r
294   *          [-o &lt;# outstanding splits&gt;]&gt;
295   *          [-D &lt;conf.param=value&gt;]
296   * @throws IOException
297   *           HBase IO problem
298   * @throws InterruptedException
299   *           user requested exit
300   * @throws ParseException
301   *           problem parsing user input
302   */
303  @SuppressWarnings("static-access")
304  public static void main(String[] args) throws IOException,
305      InterruptedException, ParseException {
306    Configuration conf = HBaseConfiguration.create();
307
308    // parse user input
309    Options opt = new Options();
310    opt.addOption(OptionBuilder.withArgName("property=value").hasArg()
311        .withDescription("Override HBase Configuration Settings").create("D"));
312    opt.addOption(OptionBuilder.withArgName("region count").hasArg()
313        .withDescription(
314            "Create a new table with a pre-split number of regions")
315        .create("c"));
316    opt.addOption(OptionBuilder.withArgName("family:family:...").hasArg()
317        .withDescription(
318            "Column Families to create with new table.  Required with -c")
319        .create("f"));
320    opt.addOption("h", false, "Print this usage help");
321    opt.addOption("r", false, "Perform a rolling split of an existing region");
322    opt.addOption(OptionBuilder.withArgName("count").hasArg().withDescription(
323        "Max outstanding splits that have unfinished major compactions")
324        .create("o"));
325    opt.addOption(null, "firstrow", true,
326        "First Row in Table for Split Algorithm");
327    opt.addOption(null, "lastrow", true,
328        "Last Row in Table for Split Algorithm");
329    opt.addOption(null, "risky", false,
330        "Skip verification steps to complete quickly. "
331            + "STRONGLY DISCOURAGED for production systems.  ");
332    CommandLine cmd = new GnuParser().parse(opt, args);
333
334    if (cmd.hasOption("D")) {
335      for (String confOpt : cmd.getOptionValues("D")) {
336        String[] kv = confOpt.split("=", 2);
337        if (kv.length == 2) {
338          conf.set(kv[0], kv[1]);
339          LOG.debug("-D configuration override: " + kv[0] + "=" + kv[1]);
340        } else {
341          throw new ParseException("-D option format invalid: " + confOpt);
342        }
343      }
344    }
345
346    if (cmd.hasOption("risky")) {
347      conf.setBoolean("split.verify", false);
348    }
349
350    boolean createTable = cmd.hasOption("c") && cmd.hasOption("f");
351    boolean rollingSplit = cmd.hasOption("r");
352    boolean oneOperOnly = createTable ^ rollingSplit;
353
354    if (2 != cmd.getArgList().size() || !oneOperOnly || cmd.hasOption("h")) {
355      new HelpFormatter().printHelp("bin/hbase regionsplitter <TABLE> <SPLITALGORITHM>\n"+
356          "SPLITALGORITHM is the java class name of a class implementing " +
357          "SplitAlgorithm, or one of the special strings HexStringSplit or " +
358          "DecimalStringSplit or UniformSplit, which are built-in split algorithms. " +
359          "HexStringSplit treats keys as hexadecimal ASCII, and " +
360          "DecimalStringSplit treats keys as decimal ASCII, and " +
361          "UniformSplit treats keys as arbitrary bytes.", opt);
362      return;
363    }
364    TableName tableName = TableName.valueOf(cmd.getArgs()[0]);
365    String splitClass = cmd.getArgs()[1];
366    SplitAlgorithm splitAlgo = newSplitAlgoInstance(conf, splitClass);
367
368    if (cmd.hasOption("firstrow")) {
369      splitAlgo.setFirstRow(cmd.getOptionValue("firstrow"));
370    }
371    if (cmd.hasOption("lastrow")) {
372      splitAlgo.setLastRow(cmd.getOptionValue("lastrow"));
373    }
374
375    if (createTable) {
376      conf.set("split.count", cmd.getOptionValue("c"));
377      createPresplitTable(tableName, splitAlgo, cmd.getOptionValue("f").split(":"), conf);
378    }
379
380    if (rollingSplit) {
381      if (cmd.hasOption("o")) {
382        conf.set("split.outstanding", cmd.getOptionValue("o"));
383      }
384      rollingSplit(tableName, splitAlgo, conf);
385    }
386  }

388  static void createPresplitTable(TableName tableName, SplitAlgorithm splitAlgo,
389          String[] columnFamilies, Configuration conf)
390  throws IOException, InterruptedException {
391    final int splitCount = conf.getInt("split.count", 0);
392    Preconditions.checkArgument(splitCount > 1, "Split count must be > 1");
393
394    Preconditions.checkArgument(columnFamilies.length > 0,
395        "Must specify at least one column family. ");
396    LOG.debug("Creating table " + tableName + " with " + columnFamilies.length
397        + " column families.  Presplitting to " + splitCount + " regions");
398
399    TableDescriptorBuilder builder = TableDescriptorBuilder.newBuilder(tableName);
400    for (String cf : columnFamilies) {
401      builder.setColumnFamily(ColumnFamilyDescriptorBuilder.of(cf));
402    }
403    try (Connection connection = ConnectionFactory.createConnection(conf)) {
404      Admin admin = connection.getAdmin();
405      try {
406        Preconditions.checkArgument(!admin.tableExists(tableName),
407          "Table already exists: " + tableName);
408        admin.createTable(builder.build(), splitAlgo.split(splitCount));
409      } finally {
410        admin.close();
411      }
412      LOG.debug("Table created!  Waiting for regions to show online in META...");
413      if (!conf.getBoolean("split.verify", true)) {
414        // NOTE: createTable is synchronous on the table, but not on the regions
415        int onlineRegions = 0;
416        while (onlineRegions < splitCount) {
417          onlineRegions = MetaTableAccessor.getRegionCount(connection, tableName);
418          LOG.debug(onlineRegions + " of " + splitCount + " regions online...");
419          if (onlineRegions < splitCount) {
420            Thread.sleep(10 * 1000); // sleep
421          }
422        }
423      }
424      LOG.debug("Finished creating table with " + splitCount + " regions");
425    }
426  }

441  private static byte [] readFile(final FileSystem fs, final Path path) throws IOException {
442    FSDataInputStream tmpIn = fs.open(path);
443    try {
444      byte [] rawData = new byte[tmpIn.available()];
445      tmpIn.readFully(rawData);
446      return rawData;
447    } finally {
448      tmpIn.close();
449    }

452  static void rollingSplit(TableName tableName, SplitAlgorithm splitAlgo, Configuration conf)
453  throws IOException, InterruptedException {
454    final int minOS = conf.getInt("split.outstanding", 2);
455    try (Connection connection = ConnectionFactory.createConnection(conf)) {
456      // Max outstanding splits. default == 50% of servers
457      final int MAX_OUTSTANDING = Math.max(getRegionServerCount(connection) / 2, minOS);
458
459      Path hbDir = FSUtils.getRootDir(conf);
460      Path tableDir = FSUtils.getTableDir(hbDir, tableName);
461      Path splitFile = new Path(tableDir, "_balancedSplit");
462      FileSystem fs = FileSystem.get(conf);
463
464      // Get a list of daughter regions to create
465      LinkedList<Pair<byte[], byte[]>> tmpRegionSet = null;
466      try (Table table = connection.getTable(tableName)) {
467        tmpRegionSet = getSplits(connection, tableName, splitAlgo);
468      }
469      LinkedList<Pair<byte[], byte[]>> outstanding = Lists.newLinkedList();
470      int splitCount = 0;
471      final int origCount = tmpRegionSet.size();
472
473      // all splits must compact & we have 1 compact thread, so 2 split
474      // requests to the same RS can stall the outstanding split queue.
475      // To fix, group the regions into an RS pool and round-robin through it
476      LOG.debug("Bucketing regions by regionserver...");
477      TreeMap<ServerName, LinkedList<Pair<byte[], byte[]>>> daughterRegions =
478          Maps.newTreeMap();
479      // Get a regionLocator.  Need it in below.
480      try (RegionLocator regionLocator = connection.getRegionLocator(tableName)) {
481        for (Pair<byte[], byte[]> dr : tmpRegionSet) {
482          ServerName rsLocation = regionLocator.getRegionLocation(dr.getSecond()).getServerName();
483          if (!daughterRegions.containsKey(rsLocation)) {
484            LinkedList<Pair<byte[], byte[]>> entry = Lists.newLinkedList();
485            daughterRegions.put(rsLocation, entry);
486          }
487          daughterRegions.get(rsLocation).add(dr);
488        }
489        LOG.debug("Done with bucketing.  Split time!");
490        long startTime = System.currentTimeMillis();
491
492        // Open the split file and modify it as splits finish
493        byte[] rawData = readFile(fs, splitFile);
494
495        FSDataOutputStream splitOut = fs.create(splitFile);
496        try {
497          splitOut.write(rawData);
498
499          try {
500            // *** split code ***
501            while (!daughterRegions.isEmpty()) {
502              LOG.debug(daughterRegions.size() + " RS have regions to splt.");
503
504              // Get ServerName to region count mapping
505              final TreeMap<ServerName, Integer> rsSizes = Maps.newTreeMap();
506              List<HRegionLocation> hrls = regionLocator.getAllRegionLocations();
507              for (HRegionLocation hrl: hrls) {
508                ServerName sn = hrl.getServerName();
509                if (rsSizes.containsKey(sn)) {
510                  rsSizes.put(sn, rsSizes.get(sn) + 1);
511                } else {
512                  rsSizes.put(sn, 1);
513                }
514              }
515
516              // Round-robin through the ServerName list. Choose the lightest-loaded servers
517              // first to keep the master from load-balancing regions as we split.
518              for (Map.Entry<ServerName, LinkedList<Pair<byte[], byte[]>>> daughterRegion :
519                      daughterRegions.entrySet()) {
520                Pair<byte[], byte[]> dr = null;
521                ServerName rsLoc = daughterRegion.getKey();
522                LinkedList<Pair<byte[], byte[]>> regionList = daughterRegion.getValue();
523
524                // Find a region in the ServerName list that hasn't been moved
525                LOG.debug("Finding a region on " + rsLoc);
526                while (!regionList.isEmpty()) {
527                  dr = regionList.pop();
528
529                  // get current region info
530                  byte[] split = dr.getSecond();
531                  HRegionLocation regionLoc = regionLocator.getRegionLocation(split);
532
533                  // if this region moved locations
534                  ServerName newRs = regionLoc.getServerName();
535                  if (newRs.compareTo(rsLoc) != 0) {
536                    LOG.debug("Region with " + splitAlgo.rowToStr(split)
537                        + " moved to " + newRs + ". Relocating...");
538                    // relocate it, don't use it right now
539                    if (!daughterRegions.containsKey(newRs)) {
540                      LinkedList<Pair<byte[], byte[]>> entry = Lists.newLinkedList();
541                      daughterRegions.put(newRs, entry);
542                    }
543                    daughterRegions.get(newRs).add(dr);
544                    dr = null;
545                    continue;
546                  }
547
548                  // make sure this region wasn't already split
549                  byte[] sk = regionLoc.getRegion().getStartKey();
550                  if (sk.length != 0) {
551                    if (Bytes.equals(split, sk)) {
552                      LOG.debug("Region already split on "
553                          + splitAlgo.rowToStr(split) + ".  Skipping this region...");
554                      ++splitCount;
555                      dr = null;
556                      continue;
557                    }
558                    byte[] start = dr.getFirst();
559                    Preconditions.checkArgument(Bytes.equals(start, sk), splitAlgo
560                        .rowToStr(start) + " != " + splitAlgo.rowToStr(sk));
561                  }
562
563                  // passed all checks! found a good region
564                  break;
565                }
566                if (regionList.isEmpty()) {
567                  daughterRegions.remove(rsLoc);
568                }
569                if (dr == null)
570                  continue;
571
572                // we have a good region, time to split!
573                byte[] split = dr.getSecond();
574                LOG.debug("Splitting at " + splitAlgo.rowToStr(split));
575                try (Admin admin = connection.getAdmin()) {
576                  admin.split(tableName, split);
577                }
578
579                LinkedList<Pair<byte[], byte[]>> finished = Lists.newLinkedList();
580                LinkedList<Pair<byte[], byte[]>> local_finished = Lists.newLinkedList();
581                if (conf.getBoolean("split.verify", true)) {
582                  // we need to verify and rate-limit our splits
583                  outstanding.addLast(dr);
584                  // with too many outstanding splits, wait for some to finish
585                  while (outstanding.size() >= MAX_OUTSTANDING) {
586                    LOG.debug("Wait for outstanding splits " + outstanding.size());
587                    local_finished = splitScan(outstanding, connection, tableName, splitAlgo);
588                    if (local_finished.isEmpty()) {
589                      Thread.sleep(30 * 1000);
590                    } else {
591                      finished.addAll(local_finished);
592                      outstanding.removeAll(local_finished);
593                      LOG.debug(local_finished.size() + " outstanding splits finished");
594                    }
595                  }
596                } else {
597                  finished.add(dr);
598                }
599
600                // mark each finished region as successfully split.
601                for (Pair<byte[], byte[]> region : finished) {
602                  splitOut.writeChars("- " + splitAlgo.rowToStr(region.getFirst())
603                      + " " + splitAlgo.rowToStr(region.getSecond()) + "\n");
604                  splitCount++;
605                  if (splitCount % 10 == 0) {
606                    long tDiff = (System.currentTimeMillis() - startTime)
607                        / splitCount;
608                    LOG.debug("STATUS UPDATE: " + splitCount + " / " + origCount
609                        + ". Avg Time / Split = "
610                        + org.apache.hadoop.util.StringUtils.formatTime(tDiff));
611                  }
612                }
613              }
614            }
615            if (conf.getBoolean("split.verify", true)) {
616              while (!outstanding.isEmpty()) {
617                LOG.debug("Finally Wait for outstanding splits " + outstanding.size());
618                LinkedList<Pair<byte[], byte[]>> finished = splitScan(outstanding,
619                    connection, tableName, splitAlgo);
620                if (finished.isEmpty()) {
621                  Thread.sleep(30 * 1000);
622                } else {
623                  outstanding.removeAll(finished);
624                  for (Pair<byte[], byte[]> region : finished) {
625                    splitOut.writeChars("- " + splitAlgo.rowToStr(region.getFirst())
626                        + " " + splitAlgo.rowToStr(region.getSecond()) + "\n");
627                    splitCount++;
628                  }
629                  LOG.debug("Finally " + finished.size() + " outstanding splits finished");
630                }
631              }
632            }
633            LOG.debug("All regions have been successfully split!");
634          } finally {
635            long tDiff = System.currentTimeMillis() - startTime;
636            LOG.debug("TOTAL TIME = "
637                + org.apache.hadoop.util.StringUtils.formatTime(tDiff));
638            LOG.debug("Splits = " + splitCount);
639            if (0 < splitCount) {
640              LOG.debug("Avg Time / Split = "
641                  + org.apache.hadoop.util.StringUtils.formatTime(tDiff / splitCount));
642            }
643          }
644        } finally {
645          splitOut.close();
646          fs.delete(splitFile, false);
647        }
648      }
649    }
650  }

656  public static SplitAlgorithm newSplitAlgoInstance(Configuration conf,
657          String splitClassName) throws IOException {
658    Class<?> splitClass;
659
660    // For split algorithms builtin to RegionSplitter, the user can specify
661    // their simple class name instead of a fully qualified class name.
662    if(splitClassName.equals(HexStringSplit.class.getSimpleName())) {
663      splitClass = HexStringSplit.class;
664    } else if (splitClassName.equals(DecimalStringSplit.class.getSimpleName())) {
665      splitClass = DecimalStringSplit.class;
666    } else if (splitClassName.equals(UniformSplit.class.getSimpleName())) {
667      splitClass = UniformSplit.class;
668    } else {
669      try {
670        splitClass = conf.getClassByName(splitClassName);
671      } catch (ClassNotFoundException e) {
672        throw new IOException("Couldn't load split class " + splitClassName, e);
673      }
674      if(splitClass == null) {
675        throw new IOException("Failed loading split class " + splitClassName);
676      }
677      if(!SplitAlgorithm.class.isAssignableFrom(splitClass)) {
678        throw new IOException(
679                "Specified split class doesn't implement SplitAlgorithm");
680      }
681    }
682    try {
683      return splitClass.asSubclass(SplitAlgorithm.class).getDeclaredConstructor().newInstance();
684    } catch (Exception e) {
685      throw new IOException("Problem loading split algorithm: ", e);
686    }
687  }

689  static LinkedList<Pair<byte[], byte[]>> splitScan(
690      LinkedList<Pair<byte[], byte[]>> regionList,
691      final Connection connection,
692      final TableName tableName,
693      SplitAlgorithm splitAlgo)
694      throws IOException, InterruptedException {
695    LinkedList<Pair<byte[], byte[]>> finished = Lists.newLinkedList();
696    LinkedList<Pair<byte[], byte[]>> logicalSplitting = Lists.newLinkedList();
697    LinkedList<Pair<byte[], byte[]>> physicalSplitting = Lists.newLinkedList();
698
699    // Get table info
700    Pair<Path, Path> tableDirAndSplitFile =
701      getTableDirAndSplitFile(connection.getConfiguration(), tableName);
702    Path tableDir = tableDirAndSplitFile.getFirst();
703    FileSystem fs = tableDir.getFileSystem(connection.getConfiguration());
704    // Clear the cache to forcibly refresh region information
705    connection.clearRegionLocationCache();
706    TableDescriptor htd = null;
707    try (Table table = connection.getTable(tableName)) {
708      htd = table.getDescriptor();
709    }
710    try (RegionLocator regionLocator = connection.getRegionLocator(tableName)) {
711      // for every region that hasn't been verified as a finished split
712      for (Pair<byte[], byte[]> region : regionList) {
713        byte[] start = region.getFirst();
714        byte[] split = region.getSecond();
715
716        // see if the new split daughter region has come online
717        try {
718          RegionInfo dri = regionLocator.getRegionLocation(split, true).getRegion();
719          if (dri.isOffline() || !Bytes.equals(dri.getStartKey(), split)) {
720            logicalSplitting.add(region);
721            continue;
722          }
723        } catch (NoServerForRegionException nsfre) {
724          // NSFRE will occur if the old hbase:meta entry has no server assigned
725          LOG.info(nsfre.toString(), nsfre);
726          logicalSplitting.add(region);
727          continue;
728        }
729
730        try {
731          // when a daughter region is opened, a compaction is triggered
732          // wait until compaction completes for both daughter regions
733          LinkedList<RegionInfo> check = Lists.newLinkedList();
734          check.add(regionLocator.getRegionLocation(start).getRegion());
735          check.add(regionLocator.getRegionLocation(split).getRegion());
736          for (RegionInfo hri : check.toArray(new RegionInfo[check.size()])) {
737            byte[] sk = hri.getStartKey();
738            if (sk.length == 0)
739              sk = splitAlgo.firstRow();
740
741            HRegionFileSystem regionFs = HRegionFileSystem.openRegionFromFileSystem(
742                connection.getConfiguration(), fs, tableDir, hri, true);
743
744            // Check every Column Family for that region -- check does not have references.
745            boolean refFound = false;
746            for (ColumnFamilyDescriptor c : htd.getColumnFamilies()) {
747              if ((refFound = regionFs.hasReferences(c.getNameAsString()))) {
748                break;
749              }
750            }
751
752            // compaction is completed when all reference files are gone
753            if (!refFound) {
754              check.remove(hri);
755            }
756          }
757          if (check.isEmpty()) {
758            finished.add(region);
759          } else {
760            physicalSplitting.add(region);
761          }
762        } catch (NoServerForRegionException nsfre) {
763          LOG.debug("No Server Exception thrown for: " + splitAlgo.rowToStr(start));
764          physicalSplitting.add(region);
765          connection.clearRegionLocationCache();
766        }
767      }
768
769      LOG.debug("Split Scan: " + finished.size() + " finished / "
770          + logicalSplitting.size() + " split wait / "
771          + physicalSplitting.size() + " reference wait");
772
773      return finished;
774    }

792  static LinkedList<Pair<byte[], byte[]>> getSplits(final Connection connection,
793      TableName tableName, SplitAlgorithm splitAlgo)
794  throws IOException {
795    Pair<Path, Path> tableDirAndSplitFile =
796      getTableDirAndSplitFile(connection.getConfiguration(), tableName);
797    Path tableDir = tableDirAndSplitFile.getFirst();
798    Path splitFile = tableDirAndSplitFile.getSecond();
799
800    FileSystem fs = tableDir.getFileSystem(connection.getConfiguration());
801
802    // Using strings because (new byte[]{0}).equals(new byte[]{0}) == false
803    Set<Pair<String, String>> daughterRegions = Sets.newHashSet();
804
805    // Does a split file exist?
806    if (!fs.exists(splitFile)) {
807      // NO = fresh start. calculate splits to make
808      LOG.debug("No " + splitFile.getName() + " file. Calculating splits ");
809
810      // Query meta for all regions in the table
811      Set<Pair<byte[], byte[]>> rows = Sets.newHashSet();
812      Pair<byte[][], byte[][]> tmp = null;
813      try (RegionLocator regionLocator = connection.getRegionLocator(tableName)) {
814        tmp = regionLocator.getStartEndKeys();
815      }
816      Preconditions.checkArgument(tmp.getFirst().length == tmp.getSecond().length,
817          "Start and End rows should be equivalent");
818      for (int i = 0; i < tmp.getFirst().length; ++i) {
819        byte[] start = tmp.getFirst()[i], end = tmp.getSecond()[i];
820        if (start.length == 0)
821          start = splitAlgo.firstRow();
822        if (end.length == 0)
823          end = splitAlgo.lastRow();
824        rows.add(Pair.newPair(start, end));
825      }
826      LOG.debug("Table " + tableName + " has " + rows.size() + " regions that will be split.");
827
828      // prepare the split file
829      Path tmpFile = new Path(tableDir, "_balancedSplit_prepare");
830      FSDataOutputStream tmpOut = fs.create(tmpFile);
831
832      // calculate all the splits == [daughterRegions] = [(start, splitPoint)]
833      for (Pair<byte[], byte[]> r : rows) {
834        byte[] splitPoint = splitAlgo.split(r.getFirst(), r.getSecond());
835        String startStr = splitAlgo.rowToStr(r.getFirst());
836        String splitStr = splitAlgo.rowToStr(splitPoint);
837        daughterRegions.add(Pair.newPair(startStr, splitStr));
838        LOG.debug("Will Split [" + startStr + " , "
839            + splitAlgo.rowToStr(r.getSecond()) + ") at " + splitStr);
840        tmpOut.writeChars("+ " + startStr + splitAlgo.separator() + splitStr
841            + "\n");
842      }
843      tmpOut.close();
844      fs.rename(tmpFile, splitFile);
845    } else {
846      LOG.debug("_balancedSplit file found. Replay log to restore state...");
847      FSUtils.getInstance(fs, connection.getConfiguration())
848        .recoverFileLease(fs, splitFile, connection.getConfiguration(), null);
849
850      // parse split file and process remaining splits
851      FSDataInputStream tmpIn = fs.open(splitFile);
852      StringBuilder sb = new StringBuilder(tmpIn.available());
853      while (tmpIn.available() > 0) {
854        sb.append(tmpIn.readChar());
855      }
856      tmpIn.close();
857      for (String line : sb.toString().split("\n")) {
858        String[] cmd = line.split(splitAlgo.separator());
859        Preconditions.checkArgument(3 == cmd.length);
860        byte[] start = splitAlgo.strToRow(cmd[1]);
861        String startStr = splitAlgo.rowToStr(start);
862        byte[] splitPoint = splitAlgo.strToRow(cmd[2]);
863        String splitStr = splitAlgo.rowToStr(splitPoint);
864        Pair<String, String> r = Pair.newPair(startStr, splitStr);
865        if (cmd[0].equals("+")) {
866          LOG.debug("Adding: " + r);
867          daughterRegions.add(r);
868        } else {
869          LOG.debug("Removing: " + r);
870          Preconditions.checkArgument(cmd[0].equals("-"),
871              "Unknown option: " + cmd[0]);
872          Preconditions.checkState(daughterRegions.contains(r),
873              "Missing row: " + r);
874          daughterRegions.remove(r);
875        }
876      }
877      LOG.debug("Done reading. " + daughterRegions.size() + " regions left.");
878    }
879    LinkedList<Pair<byte[], byte[]>> ret = Lists.newLinkedList();
880    for (Pair<String, String> r : daughterRegions) {
881      ret.add(Pair.newPair(splitAlgo.strToRow(r.getFirst()), splitAlgo
882          .strToRow(r.getSecond())));
883    }
884    return ret;
885  }

928
929    String firstRow;
930    BigInteger firstRowInt;
931    String lastRow;
932    BigInteger lastRowInt;
933    int rowComparisonLength;
934    int radix;
935
936    NumberStringSplit(String minRow, String maxRow, int radix) {
937      this.firstRow = minRow;
938      this.lastRow = maxRow;
939      this.radix = radix;
940      this.firstRowInt = BigInteger.ZERO;
941      this.lastRowInt = new BigInteger(lastRow, this.radix);
942      this.rowComparisonLength = lastRow.length();

945    @Override
946    public byte[] split(byte[] start, byte[] end) {
947      BigInteger s = convertToBigInteger(start);
948      BigInteger e = convertToBigInteger(end);
949      Preconditions.checkArgument(!e.equals(BigInteger.ZERO));
950      return convertToByte(split2(s, e));
951    }

953    @Override
954    public byte[][] split(int n) {
955      Preconditions.checkArgument(lastRowInt.compareTo(firstRowInt) > 0,
956          "last row (%s) is configured less than first row (%s)", lastRow,
957          firstRow);
958      // +1 to range because the last row is inclusive
959      BigInteger range = lastRowInt.subtract(firstRowInt).add(BigInteger.ONE);
960      Preconditions.checkState(range.compareTo(BigInteger.valueOf(n)) >= 0,
961          "split granularity (%s) is greater than the range (%s)", n, range);
962
963      BigInteger[] splits = new BigInteger[n - 1];
964      BigInteger sizeOfEachSplit = range.divide(BigInteger.valueOf(n));
965      for (int i = 1; i < n; i++) {
966        // NOTE: this means the last region gets all the slop.
967        // This is not a big deal if we're assuming n << MAXHEX
968        splits[i - 1] = firstRowInt.add(sizeOfEachSplit.multiply(BigInteger
969            .valueOf(i)));
970      }
971      return convertToBytes(splits);
972    }
973
974    @Override
975    public byte[][] split(byte[] start, byte[] end, int numSplits, boolean inclusive) {
976      BigInteger s = convertToBigInteger(start);
977      BigInteger e = convertToBigInteger(end);
978
979      Preconditions.checkArgument(e.compareTo(s) > 0,
980                      "last row (%s) is configured less than first row (%s)", rowToStr(end),
981                      end);
982      // +1 to range because the last row is inclusive
983      BigInteger range = e.subtract(s).add(BigInteger.ONE);
984      Preconditions.checkState(range.compareTo(BigInteger.valueOf(numSplits)) >= 0,
985              "split granularity (%s) is greater than the range (%s)", numSplits, range);
986
987      BigInteger[] splits = new BigInteger[numSplits - 1];
988      BigInteger sizeOfEachSplit = range.divide(BigInteger.valueOf(numSplits));
989      for (int i = 1; i < numSplits; i++) {
990        // NOTE: this means the last region gets all the slop.
991        // This is not a big deal if we're assuming n << MAXHEX
992        splits[i - 1] = s.add(sizeOfEachSplit.multiply(BigInteger
993                .valueOf(i)));
994      }
995
996      if (inclusive) {
997        BigInteger[] inclusiveSplitPoints = new BigInteger[numSplits + 1];
998        inclusiveSplitPoints[0] = convertToBigInteger(start);
999        inclusiveSplitPoints[numSplits] = convertToBigInteger(end);
1000        System.arraycopy(splits, 0, inclusiveSplitPoints, 1, splits.length);
1001        return convertToBytes(inclusiveSplitPoints);
1002      } else {
1003        return convertToBytes(splits);
1004      }
1005    }

1123   * A SplitAlgorithm that divides the space of possible keys evenly. Useful
1124   * when the keys are approximately uniform random bytes (e.g. hashes). Rows
1125   * are raw byte values in the range <b>00 =&gt; FF</b> and are right-padded with
1126   * zeros to keep the same memcmp() order. This is the natural algorithm to use
1127   * for a byte[] environment and saves space, but is not necessarily the
1128   * easiest for readability.
1129   */
1130  public static class UniformSplit implements SplitAlgorithm {
1131    static final byte xFF = (byte) 0xFF;
1132    byte[] firstRowBytes = ArrayUtils.EMPTY_BYTE_ARRAY;
1133    byte[] lastRowBytes =
1134            new byte[] {xFF, xFF, xFF, xFF, xFF, xFF, xFF, xFF};
1135    @Override
1136    public byte[] split(byte[] start, byte[] end) {
1137      return Bytes.split(start, end, 1)[1];
1138    }

1140    @Override
1141    public byte[][] split(int numRegions) {
1142      Preconditions.checkArgument(
1143          Bytes.compareTo(lastRowBytes, firstRowBytes) > 0,
1144          "last row (%s) is configured less than first row (%s)",
1145          Bytes.toStringBinary(lastRowBytes),
1146          Bytes.toStringBinary(firstRowBytes));
1147
1148      byte[][] splits = Bytes.split(firstRowBytes, lastRowBytes, true,
1149          numRegions - 1);
1150      Preconditions.checkState(splits != null,
1151          "Could not split region with given user input: " + this);
1152
1153      // remove endpoints, which are included in the splits list
1154
1155      return splits == null? null: Arrays.copyOfRange(splits, 1, splits.length - 1);
1156    }

1158    @Override

1161        start = firstRowBytes;
1162      }
1163      if (Arrays.equals(end, HConstants.EMPTY_BYTE_ARRAY)) {
1164        end = lastRowBytes;
1165      }
1166      Preconditions.checkArgument(
1167              Bytes.compareTo(end, start) > 0,
1168              "last row (%s) is configured less than first row (%s)",
1169              Bytes.toStringBinary(end),
1170              Bytes.toStringBinary(start));
1171
1172      byte[][] splits = Bytes.split(start, end, true,
1173              numSplits - 1);
1174      Preconditions.checkState(splits != null,
1175              "Could not calculate input splits with given user input: " + this);
1176      if (inclusive) {
1177        return splits;
1178      } else {
1179        // remove endpoints, which are included in the splits list
1180        return Arrays.copyOfRange(splits, 1, splits.length - 1);
1181      }
1182    }

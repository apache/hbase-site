031/**
032 * This class defines methods that can help with managing HBase clusters
033 * from unit tests and system tests. There are 3 types of cluster deployments:
034 * <ul>
035 * <li><b>MiniHBaseCluster:</b> each server is run in the same JVM in separate threads,
036 * used by unit tests</li>
037 * <li><b>DistributedHBaseCluster:</b> the cluster is pre-deployed, system and integration tests can
038 * interact with the cluster. </li>
039 * <li><b>ProcessBasedLocalHBaseCluster:</b> each server is deployed locally but in separate
040 * JVMs. </li>
041 * </ul>
042 * <p>
043 * HBaseCluster unifies the way tests interact with the cluster, so that the same test can
044 * be run against a mini-cluster during unit test execution, or a distributed cluster having
045 * tens/hundreds of nodes during execution of integration tests.
046 *
047 * <p>
048 * HBaseCluster exposes client-side public interfaces to tests, so that tests does not assume
049 * running in a particular mode. Not all the tests are suitable to be run on an actual cluster,
050 * and some tests will still need to mock stuff and introspect internal state. For those use
051 * cases from unit tests, or if more control is needed, you can use the subclasses directly.
052 * In that sense, this class does not abstract away <strong>every</strong> interface that
053 * MiniHBaseCluster or DistributedHBaseCluster provide.
054 */

126  /**
127   * Wait for the specified region server to join the cluster
128   * @throws IOException if something goes wrong or timeout occurs
129   */
130  public void waitForRegionServerToStart(String hostname, int port, long timeout)
131      throws IOException {
132    long start = EnvironmentEdgeManager.currentTime();
133    while ((EnvironmentEdgeManager.currentTime() - start) < timeout) {
134      for (ServerName server : getClusterMetrics().getLiveServerMetrics().keySet()) {
135        if (server.getHostname().equals(hostname) && server.getPort() == port) {
136          return;
137        }
138      }
139      Threads.sleep(100);
140    }
141    throw new IOException("did timeout " + timeout + "ms waiting for region server to start: "
142        + hostname);
143  }
